# おうち当番

家族2人で使う「ゴミカレンダー＋家事当番＋AI相談」統合PWAアプリです。

## 機能

- **今日の表示**: 今日のゴミ種別と担当家事を一目で確認
- **ゴミカレンダー**: 西宮市のゴミ収集スケジュールを自動取得・表示
- **家事ポイント**: スキップポイントで公平な分担を自動調整
- **AI相談**: Claude AIに家事分担の相談ができるチャット機能
- **メール通知**: 毎朝の担当家事をメールでお知らせ

## セットアップ手順

### 1. Googleスプレッドシートの作成

1. [Google スプレッドシート](https://sheets.google.com) で新しいスプレッドシートを作成
2. 以下の3つのシートを作成（シート名は正確に入力してください）:

**シート「当番表」**
| A列 | B列 | C列 | D列 | E列 |
|------|------|------|------|------|
| 日付 | 家事名 | 担当者 | 完了 | スキップ |

**シート「ポイント」**
| A列 | B列 | C列 |
|------|------|------|
| メンバー名 | スキップポイント累計 | 完了数累計 |

**シート「設定」**
| A列 | B列 | C列 |
|------|------|------|
| メンバー名 | メールアドレス | 通知時刻 |

3. スプレッドシートの共有設定を「リンクを知っている全員が閲覧可」に変更
4. URLからスプレッドシートIDをコピー
   - URL例: `https://docs.google.com/spreadsheets/d/XXXXXXXX/edit`
   - `XXXXXXXX` の部分がスプレッドシートID

### 2. Google Sheets APIキーの取得

1. [Google Cloud Console](https://console.cloud.google.com) にアクセス
2. 新しいプロジェクトを作成（または既存のプロジェクトを選択）
3. 「APIとサービス」→「ライブラリ」で「Google Sheets API」を検索し有効化
4. 「APIとサービス」→「認証情報」→「認証情報を作成」→「APIキー」
5. 作成されたAPIキーをコピー
6. （推奨）APIキーの制限を設定:
   - アプリケーションの制限: HTTP リファラー
   - APIの制限: Google Sheets API のみ

### 3. Claude APIキーの取得

1. [Anthropic Console](https://console.anthropic.com) にアクセス
2. アカウントを作成またはログイン
3. 「API Keys」ページで新しいAPIキーを作成
4. APIキーをコピー（このキーは再表示されないので必ず保存）

### 4. GASプロジェクトの作成

1. [Google Apps Script](https://script.google.com) で新しいプロジェクトを作成
2. デフォルトの `コード.gs` の内容をすべて削除
3. `gas/Code.gs` の内容を貼り付け
4. 「プロジェクトの設定」でスプレッドシートをコンテナバインドに設定するか、
   スクリプト内の `getSpreadsheet()` 関数を以下のように変更:
   ```javascript
   function getSpreadsheet() {
     return SpreadsheetApp.openById('あなたのスプレッドシートID');
   }
   ```

#### GAS Web Appのデプロイ

1. 「デプロイ」→「新しいデプロイ」
2. 種類: 「ウェブアプリ」を選択
3. 設定:
   - 説明: 「おうち当番API」
   - 実行ユーザー: 「自分」
   - アクセスできるユーザー: 「全員」
4. 「デプロイ」をクリック
5. 表示されたウェブアプリURLをコピー（設定画面で使用）

### 5. GASトリガーの設定（毎朝メール通知）

1. GASエディタで「トリガー」（時計アイコン）をクリック
2. 「トリガーを追加」
3. 設定:
   - 実行する関数: `sendDailyEmail`
   - イベントのソース: 時間主導型
   - 時間ベースのトリガーのタイプ: 日付ベースのタイマー
   - 時刻: 午前7時〜8時
4. 保存

### 6. GitHub Pagesへのデプロイ

1. このリポジトリをGitHubにプッシュ
2. リポジトリの「Settings」→「Pages」
3. Source: 「Deploy from a branch」を選択
4. Branch: `main`（または使用中のブランチ）、フォルダ: `/home-pwa`
5. 「Save」をクリック
6. 数分後にデプロイされたURLが表示されます

### 7. スマホのホーム画面への追加

#### iOS（Safari）
1. デプロイされたURLをSafariで開く
2. 共有ボタン（□↑）をタップ
3. 「ホーム画面に追加」をタップ
4. 「追加」をタップ

#### Android（Chrome）
1. デプロイされたURLをChromeで開く
2. メニュー（⋮）をタップ
3. 「ホーム画面に追加」をタップ
4. 「追加」をタップ

## アプリの初期設定

1. アプリを開くと設定画面が自動的に表示されます
2. 以下を入力:
   - メンバーA・Bの名前（例: たろう、はなこ）
   - メールアドレス（毎朝のメール通知用、省略可）
   - スプレッドシートID（URLの `/d/` と `/edit` の間の文字列）
   - Google Sheets APIキー（`AIza...` で始まる文字列）
   - GAS Web App URL（`https://script.google.com/macros/s/.../exec` 形式）
   - Claude APIキー（`sk-ant-...` で始まる文字列、AI相談機能用、省略可）
   - 当番開始日（今日の日付を選択）
3. 「保存」をタップ
4. 「4週間分の当番を生成」をタップ → 当番表がスプレッドシートに書き込まれます

> **最低限必要な設定**: メンバー名2つ + 当番開始日があれば、オフラインでもローカルで動作します。
> スプレッドシート連携やAI相談は、対応するAPIキーを設定した場合のみ有効になります。

## 日常の使い方

### 毎日やること

1. **アプリを開く** → 「今日」タブが表示されます
2. **ゴミを確認** → ゴミ収集日なら種別が色付きタグで表示されます（例: 🟤もやすごみ）
3. **家事を実行** → 担当家事が一覧表示されています
4. **完了をタップ** → 緑の「完了」ボタンを押すと、その家事に完了マーク（✓）が付きます
5. **できない場合は「スキップ」** → オレンジの「スキップ」ボタンを押すと、スキップポイントが+1されます

### 週次でやること

- 特に操作は不要です。ローテーションは自動で切り替わります
- ポイントタブで2人の偏りを確認できます

### 4週間ごとにやること

- 設定タブで「4週間分の当番を生成」を再度タップ
- 新しい4週間分の当番が上書き生成されます

## 各タブの使い方

### タブ1「今日」（起動時に表示）

アプリを開いたとき最初に見える画面です。

- **日付表示**: 今日の日付と曜日が大きく表示されます
- **ゴミセクション**: 今日が収集日のゴミ種別を色付きタグで表示
  - ゴミがない日は「今日はゴミなし」と表示
  - 色分け: もやすごみ(赤茶) / 不燃ごみ(青) / 資源(緑) / びん(ピンク) / プラ資源(オレンジ) / 缶・ペット(エメラルド)
- **家事セクション**: 今日の担当家事が一覧で表示
  - 各家事に「完了」「スキップ」の2ボタン
  - **完了**: タップすると家事に取り消し線が入り、スプレッドシートのD列が1になります
  - **スキップ**: タップするとスキップ扱いになり、担当者のポイントが+1されます
- **通知バナー**: 初回アクセス時、「通知を有効にする」バナーが表示されます。タップするとブラウザ通知の許可を求めます

### タブ2「ゴミカレンダー」

西宮市のゴミ収集スケジュールを週単位で表示します。

- **週間表示**: 月曜〜日曜の7日間を横並びで表示
- **ナビゲーション**:
  - 「◀ 前週」「次週 ▶」: 1週間ずつ移動
  - 「◀ 前月」「次月 ▶」: 約4週間ずつ移動
  - 「今週」: 現在の週に戻る
- **今日のセル**: 青い枠線で強調されます
- **データ取得**: 西宮市のウェブサイトからCORSプロキシ経由で自動取得し、localStorageにキャッシュされます

### タブ3「ポイント・履歴」

2人の公平さを数値で可視化する画面です。

- **スキップポイント**: 2人のポイントを並べて大きく表示
  - スキップするたびに+1ポイント
  - ポイントが多い = サボりがち、という意味
- **自動調整アラート**: ポイント差が3以上になると黄色い警告が表示され、次回の当番生成時に自動で調整されます（ポイントの多い方の担当を1つ減らす）
- **履歴一覧**: 過去2週間の完了・スキップ履歴
  - 緑背景: 完了した家事
  - オレンジ背景: スキップした家事
  - 日付、家事名、担当者が表示されます

### タブ4「設定」

各種APIキーやメンバー情報を管理する画面です。

- **メンバー設定**: 名前（表示名）とメールアドレス
- **API設定**:
  - スプレッドシートID: Googleスプレッドシートを特定するID
  - Google Sheets APIキー: データ読み取り用
  - GAS Web App URL: データ書き込み用（GASをデプロイして取得）
  - Claude APIキー: AI相談機能用（省略可）
- **当番開始日**: ローテーションの起点となる日付
- **「保存」ボタン**: 全設定をlocalStorageに保存 + GAS経由でスプレッドシートの設定シートにも保存
- **「4週間分の当番を生成」ボタン**: 開始日から4週間分の家事割り振りを計算し、スプレッドシートに書き込みます

### タブ5「AI相談」

Claude AIに家事分担について相談できるチャット画面です。

- **チャット形式**: 吹き出しで会話が表示されます
  - あなたのメッセージ: 右寄せ・薄青背景
  - AIの回答: 左寄せ・薄紫背景
- **サジェストボタン**: 画面下部に3つのボタン
  - 「今週の分担を提案して」
  - 「最近サボりがちな家事は？」
  - 「来週の調整案を出して」
  - タップすると入力欄にテキストが自動入力されます
- **AIの知識**: 毎回のリクエストに以下の情報が自動で渡されます
  - 過去4週間の家事完了・スキップ履歴
  - 各メンバーのスキップポイント累計
  - 今週の担当割り振り
- **会話のリセット**: ブラウザをリロードすると会話履歴はリセットされます（セッション中のみ保持）
- **料金**: Claude Haiku モデルを使用するため、1回の会話あたり約0.01〜0.05円程度です

## 管理する家事（7種類）

| # | 家事名 | 内容例 |
|---|--------|--------|
| 1 | ゴミ出し | 収集日にゴミを出す |
| 2 | 食事作り | 朝食・夕食の準備 |
| 3 | 食器洗い | 食後の食器洗い・片付け |
| 4 | 風呂掃除 | 浴槽・浴室の掃除 |
| 5 | トイレ掃除 | トイレの清掃 |
| 6 | 床掃除・掃除機 | フロアの掃除機がけ・拭き掃除 |
| 7 | 飲み物・日用品買い出し | 消耗品の買い出し |

> 家事の種類を変更したい場合は、`app.js` の先頭にある `CHORES` 配列を直接編集してください。

## ローテーションルール

- 7種の家事を**週単位**で2人に交互に割り振り
- **奇数週**（1週目、3週目）: メンバーAが4つ、メンバーBが3つ担当
- **偶数週**（2週目、4週目）: メンバーBが4つ、メンバーAが3つ担当
- **自動調整**: スキップポイント差が3以上になると、ポイントの多い方の担当を1つ減らして公平性を保ちます

### 割り振りの例

```
1週目（奇数週）:
  たろう: ゴミ出し、食事作り、食器洗い、風呂掃除
  はなこ: トイレ掃除、床掃除・掃除機、飲み物・日用品買い出し

2週目（偶数週）:
  はなこ: ゴミ出し、食事作り、食器洗い、風呂掃除
  たろう: トイレ掃除、床掃除・掃除機、飲み物・日用品買い出し
```

## GASセットアップ詳細

### スプレッドシートバインド型 vs スタンドアロン型

GASプロジェクトの作成方法は2通りあります:

**方法A: スタンドアロン型（推奨）**
1. [script.google.com](https://script.google.com) で「新しいプロジェクト」を作成
2. `Code.gs` を貼り付け
3. `getSpreadsheet()` 関数を以下のように変更:
   ```javascript
   function getSpreadsheet() {
     return SpreadsheetApp.openById('あなたのスプレッドシートID');
   }
   ```
4. デプロイして Web App URL を取得

**方法B: コンテナバインド型**
1. 対象のスプレッドシートを開く
2. メニュー「拡張機能」→「Apps Script」
3. `Code.gs` を貼り付け（`getSpreadsheet()` の変更は不要）
4. デプロイして Web App URL を取得

### デプロイ手順の詳細

1. GASエディタ右上の「デプロイ」→「新しいデプロイ」をクリック
2. 左上の歯車アイコンで「ウェブアプリ」を選択
3. 以下を設定:
   - **説明**: 任意（例: 「おうち当番API v1」）
   - **ウェブアプリ > 次のユーザーとして実行**: 「自分」
   - **アクセスできるユーザー**: 「全員」
4. 「デプロイ」をクリック
5. **初回のみ**: Googleアカウントの権限承認画面が表示されます
   - 「詳細」→「おうち当番API（安全ではないページ）に移動」→「許可」をクリック
   - これはGASがスプレッドシートにアクセスする権限を求めるものです
6. 表示された「ウェブアプリ URL」をコピーし、アプリの設定画面に貼り付けます

> **コードを修正した場合**: 「デプロイ」→「デプロイを管理」→右上の鉛筆アイコン→バージョン「新しいバージョン」→「デプロイ」で更新できます。

### GASトリガーの設定（毎朝メール通知）

1. GASエディタ左の時計アイコン「トリガー」をクリック
2. 右下の「+ トリガーを追加」
3. 以下を設定:
   - **実行する関数**: `sendDailyEmail`
   - **実行するデプロイ**: 「Head」
   - **イベントのソース**: 「時間主導型」
   - **時間ベースのトリガーのタイプ**: 「日付ベースのタイマー」
   - **時刻**: 「午前7時〜8時」
4. 「保存」をクリック

## トラブルシューティング

### ゴミカレンダーが表示されない
- **原因**: CORSプロキシが一時的に利用できない場合があります
- **対処**: アプリを再読み込みすると、別のプロキシで再取得を試みます
- **確認**: ブラウザのコンソール（F12→Console）にエラーが表示されていないか確認

### スプレッドシートに反映されない
- **確認1**: 設定タブの「GAS Web App URL」が正しいか確認（`https://script.google.com/macros/s/...` 形式）
- **確認2**: GASのデプロイが「全員」アクセス可能になっているか確認
- **確認3**: GASのコード修正後、新しいバージョンとしてデプロイし直しているか確認
- **確認4**: スプレッドシートIDが `getSpreadsheet()` に正しく設定されているか確認

### AI相談が動かない
- **確認1**: Claude APIキーが正しく入力されているか確認（`sk-ant-` で始まる）
- **確認2**: APIキーの残高が残っているか [console.anthropic.com](https://console.anthropic.com) で確認
- **エラー表示**: 「API接続エラー」と表示される場合、ブラウザの拡張機能（広告ブロッカー等）が干渉している可能性があります

### 通知が来ない
- **iOS**: Safari でPWAとしてホーム画面に追加した場合、iOS 16.4以降でのみ通知がサポートされます。それ以前のバージョンでは通知は利用できません
- **Android**: Chrome の設定で通知が許可されているか確認してください
- **PC**: ブラウザの設定 → サイトの権限 → 通知 で許可されているか確認

### 設定が消えた
- **原因**: ブラウザのキャッシュ/データをクリアすると localStorage も削除されます
- **対処**: スプレッドシートにデータが残っていれば、設定を再入力後に自動同期されます

## FAQ（よくある質問）

**Q: 家事の種類を変えたい**
A: `app.js` の6行目にある `CHORES` 配列を直接編集してください。変更後はService Workerのキャッシュ更新のため、ブラウザの「サイトデータを削除」が必要です。

**Q: 3人以上で使いたい**
A: 現在は2人固定の設計です。3人以上への対応は大幅な改修が必要になります。

**Q: データのバックアップは？**
A: 当番表・ポイント・設定はすべてGoogleスプレッドシートに保存されるため、スプレッドシート自体がバックアップになります。localStorageのデータが消えても、スプレッドシートから復元されます。

**Q: 複数のデバイスで使える？**
A: はい。同じスプレッドシートIDとAPIキーを設定すれば、複数デバイスで同じデータを共有できます。

**Q: オフラインで使える？**
A: Service Workerにより、画面表示やタブ切り替えはオフラインでも動作します。ただし、スプレッドシートへの同期、ゴミカレンダーの取得、AI相談はインターネット接続が必要です。オフライン中の操作はlocalStorageに保存され、次回オンライン時にスプレッドシートと同期されます。

**Q: 費用はかかる？**
A: すべて無料で利用できます。ただし、AI相談機能を使う場合のみ Claude API の従量課金（1回あたり約0.01〜0.05円）が発生します。

## セキュリティについて

- APIキーはブラウザのlocalStorageに保存されます（サーバーには送信されません）
- ソースコードにAPIキーは含まれません
- Claude APIキーはフロントエンドから直接送信されます（家庭内利用を想定したシンプルな構成です）
- 第三者にURLを共有しても、APIキーは共有されません（各自が設定画面で入力）

## 技術スタック

- HTML / CSS / JavaScript（フレームワークなし）
- PWA（Service Worker + manifest.json）
- Google Sheets API v4（読み取り）
- Google Apps Script（書き込み + メール通知）
- Claude API（AI相談機能）
